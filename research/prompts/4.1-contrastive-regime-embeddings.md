# 4.1 Contrastive Regime Embeddings

> **Domain**: Machine Learning & Deep Learning | **Priority**: P3 | **Status**: Pending

## System Context

This research targets **llm-quant**, a systematic macro trading system managing a $100k paper portfolio across 39 tradeable assets spanning six asset classes:

- **US Equity ETFs (17)**: SPY, QQQ, IWM, DIA, VTI, IVV, XLK, XLF, XLE, XLV, XLI, XLY, XLP, XLU, XLRE, XLB, XLC
- **International Equity (3)**: VGK, EEM, EFA
- **Fixed Income (6)**: TLT, IEF, SHY, LQD, HYG, TIP
- **Commodities (3)**: GLD, SLV, USO
- **Crypto (5)**: BTC-USD, ETH-USD, SOL-USD, XRP-USD, ADA-USD
- **Forex (5)**: EURUSD=X, GBPUSD=X, USDJPY=X, AUDUSD=X, USDCHF=X

**Stack**: Python 3.12, Polars DataFrames, DuckDB persistent storage, Jinja2 prompt templates.

**Current regime classification** is a simple heuristic in `src/llm_quant/brain/context.py`. The function `build_market_context()` assembles a `MarketContext` dataclass (defined in `src/llm_quant/brain/models.py`) containing:
- `vix: float` -- latest VIX close, fetched via `_get_vix()`
- `yield_spread: float` -- TLT/SHY price ratio proxy, computed by `_get_yield_spread()`
- `spy_trend: str` -- "bullish"/"bearish"/"neutral" from SPY SMA(50) vs SMA(200), via `_get_spy_trend()`

These three scalars are combined into a single string label (`"risk_on"`, `"risk_off"`, or `"transition"`) via the `MarketRegime` enum in `src/llm_quant/brain/models.py`. There is no confidence score, no transition probability, and no memory of how long the current regime has persisted. The boundaries between regimes are hard-coded thresholds.

**Current indicators** computed in `src/llm_quant/data/indicators.py` via `compute_indicators()`:
- SMA(20), SMA(50), SMA(200), Bollinger Bands (20,2), RSI(14), MACD(12,26,9), ATR(14)

**Pipeline**: `scripts/build_context.py` fetches data, computes indicators, assembles `MarketContext`, renders the Jinja2 decision prompt (`config/prompts/trader_decision.md`), and outputs JSON. The LLM reads this context and returns a `TradingDecision` parsed by `src/llm_quant/brain/parser.py`. Trades pass through 8 risk checks in `src/llm_quant/risk/manager.py` (class `RiskManager`) before execution via `scripts/execute_decision.py`.

**Targets**: Sharpe > 0.8, max drawdown < 15%, Sortino > 1.0, Calmar > 0.5.

## Research Question

Can a contrastive self-supervised model trained on multi-asset return and volatility windows produce regime embeddings that outperform the current VIX/yield-curve/momentum heuristic in terms of regime stability, forward return calibration, and downstream portfolio performance?

Specifically:
1. Does the learned embedding space capture regime transitions earlier and with fewer false signals than the three-variable heuristic?
2. Do regime-conditional return distributions calibrated from embedding clusters improve position sizing decisions?
3. Does replacing the discrete `MarketRegime` label with a continuous embedding vector improve the LLM brain's trade decision quality?

## Hypothesis

The current heuristic discretizes a continuous latent state space with hard boundaries at arbitrary VIX levels and yield spread thresholds. This creates three failure modes: (a) rapid oscillation between labels during volatile transition periods, (b) inability to distinguish structurally different regimes that share similar VIX/yield characteristics, and (c) no information about regime persistence or transition velocity.

**Prediction**: A contrastive encoder trained on 60-day rolling windows of multi-asset returns, volatilities, and cross-correlations will produce a smooth, higher-dimensional regime manifold that:
- **Reduces false regime transitions by 40-60%** (measured as label changes reversed within 5 trading days)
- **Improves Spearman rank correlation** between regime embedding distance and forward 21-day realized Sharpe from baseline ~0.1 to target >0.25
- **Yields tighter regime-conditional return distributions** (lower entropy within clusters vs. within heuristic labels)

**Null hypothesis**: The TCN-contrastive embeddings produce clusters whose forward return distributions are statistically indistinguishable from the heuristic's regime labels (Kolmogorov-Smirnov test, p > 0.05).

## Methodology

### Phase 1: Data Preparation (Week 1)

1. **Extract feature matrix** from DuckDB via `src/llm_quant/data/indicators.py` output. For each trading day `t` and each of the 39 assets, construct a 5-feature vector:
   - Log return: `ln(close_t / close_{t-1})`
   - Realized volatility: 20-day rolling standard deviation of log returns
   - RSI(14) normalized to [0, 1]
   - MACD histogram (normalized by ATR for cross-asset comparability)
   - Volume z-score (20-day rolling)
2. **Construct sliding windows**: For each day `t`, extract the window `[t-59, t]` producing a tensor of shape `(60, 39, 5)`. Flatten the asset-feature dimension to `(60, 195)` for the TCN input.
3. **Augmentation strategy** (for SimCLR positive pairs):
   - **Temporal jitter**: Shift the window start by 1-3 days randomly
   - **Feature dropout**: Zero out 10-20% of features randomly
   - **Gaussian noise**: Add N(0, 0.01) to normalized features
   - Two augmented views of the same window form a positive pair; windows from different calendar periods are negatives.

### Phase 2: Model Architecture (Week 2)

**Encoder: Temporal Convolutional Network (TCN)**

```
Input: (batch, 60, 195)
  -> Conv1D(195, 128, kernel=3, dilation=1) + BatchNorm + ReLU
  -> Conv1D(128, 128, kernel=3, dilation=2) + BatchNorm + ReLU + Residual
  -> Conv1D(128, 128, kernel=3, dilation=4) + BatchNorm + ReLU + Residual
  -> Conv1D(128, 64, kernel=3, dilation=8) + BatchNorm + ReLU + Residual
  -> GlobalAveragePool over time dimension
  -> Linear(64, 64)  # embedding
Output: (batch, 64)
```

**Projection head** (discarded after training, per SimCLR):
```
  -> Linear(64, 128) + ReLU
  -> Linear(128, 64)
```

**Loss**: NT-Xent (Normalized Temperature-scaled Cross-Entropy Loss) with temperature tau=0.07:
```
L(i, j) = -log( exp(sim(z_i, z_j)/tau) / sum_k exp(sim(z_i, z_k)/tau) )
```
where `sim` is cosine similarity and `k` iterates over all 2N examples in the batch (including the positive pair `j`).

**Training hyperparameters**:
- Batch size: 256 pairs (512 augmented views)
- Optimizer: AdamW, lr=3e-4, weight_decay=1e-4
- Scheduler: Cosine annealing over 200 epochs
- Early stopping: patience=20 epochs on validation NT-Xent loss

### Phase 3: Clustering (Week 3)

1. **Embed all historical windows** through the trained encoder (projection head removed).
2. **Dimensionality reduction**: UMAP(n_components=3, n_neighbors=15, min_dist=0.1) for visualization; use full 64-dim embeddings for clustering.
3. **Clustering**: HDBSCAN(min_cluster_size=60, min_samples=10) on 64-dim embeddings.
   - 60-day min cluster size enforces that regimes persist at least ~3 months, consistent with macro regime persistence.
4. **Cluster validation**:
   - Silhouette score on 64-dim space (target > 0.3)
   - Davies-Bouldin index (target < 1.5)
   - Calinski-Harabasz index
5. **Label alignment**: Map HDBSCAN cluster IDs to interpretable labels by examining:
   - Mean VIX level per cluster
   - Mean cross-asset correlation per cluster
   - Mean bond/equity return spread per cluster
   - Assign names like "low-vol-equity-led", "high-vol-flight-to-quality", "rotation-transition"

### Phase 4: Evaluation (Week 4)

1. **Regime stability**: Count the number of regime label changes per year for both the heuristic and the embedding-based classification. A "false transition" is a label change that reverts within 5 trading days.
2. **Forward return calibration**: For each regime cluster, compute the distribution of forward 21-day portfolio returns (60/40 SPY/TLT benchmark). Measure Spearman correlation between embedding-space distance from cluster centroid and forward Sharpe.
3. **Portfolio backtest**:
   - Variant A: Replace `MarketRegime` enum with HDBSCAN cluster ID in the LLM prompt
   - Variant B: Append top-3 nearest historical regime transitions to the prompt (combines with Brief 4.3)
   - Variant C: Use regime embedding distance to modulate position sizing (closer to "risk_off" centroid = smaller positions)
4. **Statistical significance**: Deflated Sharpe Ratio (Bailey & Lopez de Prado, 2014) to account for multiple testing across the three variants.

## Literature Review

### Core Papers

1. **Bai, Kolter & Koltun (2018)** -- "An Empirical Evaluation of Generic Convolutional and Recurrent Networks for Sequence Modeling." *arXiv:1803.01271*.
   - **Extract**: TCN architecture design (dilated causal convolutions, residual connections). Demonstrates TCN matches or beats LSTMs on standard sequence benchmarks with better parallelism and stable gradients. Use their recommended kernel size and dilation schedule.

2. **Chen, Kornblith, Norouzi & Hinton (2020)** -- "A Simple Framework for Contrastive Learning of Visual Representations (SimCLR)." *ICML 2020*.
   - **Extract**: NT-Xent loss formulation, importance of projection head (discard after training), augmentation strategy design, temperature parameter sensitivity. Their finding that larger batch sizes improve contrastive learning is critical given our small dataset.

3. **McInnes, Healy & Melville (2018)** -- "UMAP: Uniform Manifold Approximation and Projection for Dimension Reduction." *arXiv:1802.03426*.
   - **Extract**: UMAP hyperparameter guidance (n_neighbors controls local vs. global structure preservation, min_dist controls cluster tightness). Use for 2D/3D visualization of the 64-dim embedding space.

4. **Lopez de Prado (2018)** -- *Advances in Financial Machine Learning*, Chapters 9 (Hyper-Parameter Tuning with Cross-Validation) and 17 (Structural Breaks).
   - **Extract**: Combinatorial purged cross-validation (CPCV) to avoid leakage in time-series ML. Structural break detection methods (CUSUM, Chow test) as ground truth for regime transitions. The "triple barrier" method for labeling forward returns.

5. **Papenbrock, Schwendner, Jaeger & Krugel (2021)** -- "Matrix Evolutions: Synthetic Correlations and Explainable Machine Learning for Constructing Robust Investment Portfolios." *Journal of Portfolio Management*, 47(5).
   - **Extract**: Using correlation matrix structure as regime features. Their finding that synthetic correlation regimes have portfolio-relevant predictive power validates our approach of encoding cross-asset correlation dynamics.

### Supporting Papers

6. **Bailey & Lopez de Prado (2014)** -- "The Deflated Sharpe Ratio: Correcting for Selection Bias, Backtest Overfitting, and the Dispersion of Sharpe Ratios." *Journal of Portfolio Management*, 40(5).
   - **Extract**: Formula and implementation for the Deflated Sharpe Ratio. Required for all performance evaluations to account for the number of strategy variants tested.

7. **Campello, Moulavi & Sander (2013)** -- "Density-Based Clustering Based on Hierarchical Density Estimates (HDBSCAN)." *PAKDD 2013*.
   - **Extract**: HDBSCAN's ability to find clusters of varying density and to label points as noise. Critical for handling regime transitions (which may not belong to any clean cluster).

8. **Hamilton (1989)** -- "A New Approach to the Economic Analysis of Nonstationary Time Series and the Business Cycle." *Econometrica*, 57(2).
   - **Extract**: Hidden Markov Model (HMM) as the classical benchmark for regime switching. Use a 2-state Gaussian HMM as a comparison baseline alongside our heuristic.

## Evaluation Criteria

### Success Thresholds
- **Primary**: Deflated Sharpe Ratio of the best embedding-based variant exceeds baseline Deflated Sharpe Ratio by at least 0.05 (absolute), with the Deflated Sharpe itself above 0.95 (i.e., 95% probability the improvement is not due to overfitting)
- **Secondary**: False regime transitions per year reduced by >= 40% relative to heuristic
- **Secondary**: Silhouette score on test set embeddings > 0.25 (lower threshold than train due to distribution shift)

### Failure Conditions
- Embedding clusters do not separate forward return distributions (KS test p > 0.05 between all cluster pairs)
- Test set silhouette score < 0.15 (clusters are meaningless out-of-sample)
- Adding the embedding to the LLM context increases average decision latency by more than 500ms (model inference time)
- Deflated Sharpe Ratio < 0.5 (strong evidence of overfitting)

### Comparison Benchmarks
1. **Current heuristic**: VIX/yield-spread/SPY-trend three-variable classification
2. **2-state Gaussian HMM**: Classical regime-switching baseline (Hamilton 1989)
3. **K-means on PCA-reduced returns**: Simple linear alternative
4. **Rolling correlation structure** (Papenbrock et al.): Correlation-matrix-based regime detection

## Anti-Overfitting Warnings

### Critical Risks

1. **Small effective sample size**: With ~4,940 sliding windows and heavy temporal autocorrelation between overlapping windows, the effective independent sample count may be as low as ~80 (4,940 / 60). Contrastive learning with batch size 256 will see much of the dataset per epoch. **Mitigation**: Use non-overlapping windows (stride=20) for validation metrics. Report effective sample size alongside all statistics.

2. **Data leakage through overlapping windows**: Adjacent windows share 59 of 60 days. The encoder can trivially learn to produce similar embeddings for adjacent windows without learning anything about regime structure. **Mitigation**: Ensure negative pairs in contrastive loss are sampled from windows at least 120 days apart. Use combinatorial purged cross-validation (Lopez de Prado, Ch. 9) for any supervised evaluation.

3. **Cluster count selection bias**: HDBSCAN's `min_cluster_size` directly controls the number of discovered regimes. Testing multiple values and selecting the one with the best forward metrics is classic data snooping. **Mitigation**: Fix `min_cluster_size=60` before seeing any test results. Justify from domain knowledge (macro regimes persist months, not days). Do not tune this parameter.

4. **Augmentation information leakage**: Temporal jitter augmentation can create positive pairs that span regime transitions, confusing the encoder. **Mitigation**: Limit jitter to +/- 1 day maximum. Verify augmented pairs don't cross known structural breaks (CUSUM-detected).

5. **Survivor bias in asset universe**: Crypto assets (BTC, ETH, SOL, XRP, ADA) have much shorter histories than equities. The feature matrix will have missing values before ~2017 (BTC) or ~2020 (SOL). **Mitigation**: Train the encoder on the equity/bond/commodity subset (28 assets) for the full history, then fine-tune with all 39 assets on the shorter overlapping period. Report results separately for both configurations.

6. **Stationarity assumption**: The contrastive encoder assumes the relationship between features and regimes is stationary. A regime that looked like "risk_on" in 2010 may have different asset return characteristics than "risk_on" in 2023 (post-QE, higher rate environment). **Mitigation**: Evaluate embedding stability in rolling 2-year windows. If cluster assignments for structurally similar periods (e.g., both pre-recession expansions) diverge significantly, the model is capturing non-regime variation.

7. **Confirmation bias in cluster interpretation**: It is tempting to name clusters after examining their forward returns (HARKing). **Mitigation**: Label clusters using only concurrent features (VIX level, correlation structure, yield curve) -- never forward returns. The forward return analysis is strictly for evaluation, not labeling.
