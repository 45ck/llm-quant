# 4.3 RAG Analogical Reasoning

> **Domain**: Machine Learning & Deep Learning | **Priority**: P2 | **Status**: Pending

## System Context

This research targets **llm-quant**, a systematic macro trading system managing a $100k paper portfolio across 39 tradeable assets spanning six asset classes:

- **US Equity ETFs (17)**: SPY, QQQ, IWM, DIA, VTI, IVV, XLK, XLF, XLE, XLV, XLI, XLY, XLP, XLU, XLRE, XLB, XLC
- **International Equity (3)**: VGK, EEM, EFA
- **Fixed Income (6)**: TLT, IEF, SHY, LQD, HYG, TIP
- **Commodities (3)**: GLD, SLV, USO
- **Crypto (5)**: BTC-USD, ETH-USD, SOL-USD, XRP-USD, ADA-USD
- **Forex (5)**: EURUSD=X, GBPUSD=X, USDJPY=X, AUDUSD=X, USDCHF=X

**Stack**: Python 3.12, Polars DataFrames, DuckDB persistent storage, Jinja2 prompt templates.

**Current LLM decision pipeline in detail**:

1. `scripts/build_context.py::main()` orchestrates the context build:
   - Calls `src/llm_quant/data/fetcher.py::fetch_ohlcv()` if data is stale
   - Calls `src/llm_quant/data/indicators.py::compute_indicators()` which adds SMA(20/50/200), Bollinger Bands, RSI(14), MACD(12,26,9), ATR(14) per asset
   - Calls `src/llm_quant/brain/context.py::build_market_context()` which assembles a `MarketContext` dataclass containing:
     - Portfolio state: NAV, cash, positions (each with symbol, shares, avg_cost, current_price, P&L%, weight, stop_loss)
     - Market data: 39 assets with [close, change_pct, sma_20, sma_50, rsi_14, macd, atr_14, volume]
     - Macro: VIX (from `_get_vix()`), yield spread proxy (TLT/SHY ratio from `_get_yield_spread()`), SPY trend (from `_get_spy_trend()`)
   - Calls `src/llm_quant/brain/prompts.py::render_decision_prompt()` which renders `config/prompts/trader_decision.md` Jinja2 template
   - Outputs JSON with `system_prompt`, `decision_prompt`, `portfolio_summary`, `macro`, and `date`

2. The LLM (Claude) reads the rendered context and returns a JSON `TradingDecision` parsed by `src/llm_quant/brain/parser.py::parse_trading_decision()`, containing:
   - `market_regime`: one of `risk_on`, `risk_off`, `transition` (MarketRegime enum)
   - `regime_confidence`: float 0-1
   - `regime_reasoning`: string
   - `signals`: list of `TradeSignal(symbol, action, conviction, target_weight, stop_loss, reasoning)`
   - `portfolio_commentary`: string

3. `scripts/execute_decision.py::main()` processes the decision:
   - Loads portfolio from DuckDB via `src/llm_quant/trading/portfolio.py::Portfolio.from_db()`
   - Runs all signals through `src/llm_quant/risk/manager.py::RiskManager.filter_signals()` which applies 8 checks: position_size (2% NAV per trade), position_weight (10% equity/5% crypto/8% forex), gross_exposure (200%), net_exposure (100%), sector_concentration (30%), cash_reserve (5%), stop_loss (required), drawdown_limit (15% with 3% buffer)
   - Executes approved signals and logs trades

**The core problem this research addresses**: The LLM receives only today's snapshot. It has no memory of what happened in similar past market conditions. When facing an ambiguous regime (e.g., VIX at 22, yield curve slightly inverted, mixed sector momentum), the LLM has no empirical basis for calibrating its confidence. It cannot say "the last 3 times the market looked like this, here is what happened next." This leads to overconfident decisions in novel situations and underconfident decisions in well-precedented situations.

**Targets**: Sharpe > 0.8, max drawdown < 15%, Sortino > 1.0, Calmar > 0.5.

## Research Question

Does providing the LLM brain with retrieved historical analogues -- the 3-5 most similar past 30-day market windows along with what happened in the subsequent 5/10/21 trading days -- improve the quality and calibration of trade decisions?

Specifically:
1. Does the LLM make better regime assessments when shown concrete historical examples of similar market conditions?
2. Does the correlation between the LLM's stated conviction level and realized forward Sharpe ratio improve?
3. Does the LLM reduce overconfident trades during genuinely ambiguous regimes?
4. Does overall portfolio performance (Sharpe, Sortino, max drawdown) improve?

## Hypothesis

LLMs reason more effectively with concrete examples than with abstract descriptions (this is the fundamental insight behind few-shot prompting and in-context learning). The current system gives the LLM a snapshot of 39 assets' indicators plus three macro numbers and asks it to classify the regime and generate trade signals. The LLM must rely entirely on its parametric knowledge of market dynamics, which may be stale, incomplete, or miscalibrated.

**Prediction**: A RAG system that retrieves 3-5 historically similar 30-day windows (with subsequent outcomes) and appends them to the LLM prompt will:

1. **Improve regime assessment accuracy**: Measured by alignment between LLM-assigned regime and the regime that would have been optimal ex-post (based on forward 21-day returns). Expected improvement: from baseline ~55% accuracy to > 65%.
2. **Improve conviction calibration**: Measured by Spearman correlation between LLM's stated conviction (high/medium/low) and forward 21-day Sharpe of the executed trades. Expected improvement: from baseline ~0.05 to > 0.20.
3. **Reduce overconfidence in ambiguous regimes**: Measured by the fraction of "high conviction" trades made during periods that were objectively ambiguous (ex-post regime classification differs from LLM's ex-ante classification). Expected reduction: from baseline ~40% false-high-conviction to < 20%.
4. **Improve portfolio metrics**: Sharpe improvement of >= 0.10 over the no-RAG baseline, primarily through better position sizing (avoiding oversized positions in uncertain regimes).

**Key advantage**: This approach introduces zero trainable parameters. The FAISS index is a pure similarity lookup. The LLM's existing in-context reasoning capabilities do all the work. This makes it the lowest-risk, highest-compatibility enhancement in Domain 4. There is near-zero overfitting risk since no model is being fit to the data -- we are simply providing the LLM with more context.

**Null hypothesis**: Historical analogues do not improve LLM decision quality because (a) market dynamics are non-stationary enough that past analogues are misleading, or (b) the LLM already has sufficient parametric knowledge to reason about regimes.

## Methodology

### Phase 1: State Vector Construction (Week 1)

**Define the market state vector** that will be used for similarity search. For each trading day `t`, construct a 64-dimensional state vector by aggregating across the 39-asset universe:

1. **Cross-sectional summary statistics (20 dims)**:
   - Mean and std of 20-day returns across all assets (2)
   - Mean and std of RSI(14) across all assets (2)
   - Mean and std of MACD histogram across all assets (2)
   - Fraction of assets above SMA(50) (1)
   - Fraction of assets above SMA(200) (1)
   - Cross-sectional return dispersion: std of daily returns across assets (1)
   - Equity-bond correlation: 20-day rolling correlation of SPY vs TLT returns (1)
   - Equity-commodity correlation: 20-day rolling correlation of SPY vs GLD returns (1)
   - Credit spread proxy: HYG-LQD 20-day return differential (1)
   - Term structure proxy: TLT-SHY 20-day return differential (1)
   - Dollar strength: mean of USDJPY and USDCHF 20-day returns (1)
   - Crypto-equity correlation: 20-day rolling correlation of BTC-USD vs SPY (1)
   - Volume regime: mean volume z-score across equities (1)
   - Volatility regime: mean ATR(14)/close ratio across equities (1)
   - Mean RSI of sector ETFs (XLK, XLF, XLE, XLV, XLI, XLY, XLP, XLU, XLRE, XLB, XLC) (1)
   - RSI dispersion across sectors (1)

2. **Temporal dynamics (12 dims)**:
   - VIX level and its 5-day change (2)
   - SPY 5-day and 20-day returns (2)
   - SPY realized volatility (20-day) and its 20-day change (2)
   - TLT 5-day and 20-day returns (2)
   - GLD 5-day and 20-day returns (2)
   - HYG 5-day return (1)
   - EEM-SPY relative return 20-day (1)

3. **Regime indicators (6 dims)**:
   - VIX level (1)
   - VIX term structure: VIX vs 20-day realized vol of SPY (1)
   - Yield spread proxy (TLT/SHY ratio) (1)
   - SPY distance from 200-day SMA as % (1)
   - Breadth: fraction of the 17 US equity ETFs with positive 20-day returns (1)
   - Momentum factor: top-quintile minus bottom-quintile 20-day returns (1)

4. **Rolling window aggregation (26 dims)**:
   - 30-day rolling averages of all 20 cross-sectional statistics (20)
   - 30-day rolling averages of VIX, SPY return, TLT return, GLD return, HYG return, breadth (6)

**Normalization**: Each of the 64 dimensions is standardized using an expanding-window z-score (mean and std computed using only data up to day `t`, never future data). Clip to [-3, 3].

### Phase 2: FAISS Index Construction (Week 2)

1. **Compute state vectors** for every trading day in the database (~5,000 days).
2. **Build FAISS index**: Use `faiss.IndexFlatIP` (inner product on L2-normalized vectors = cosine similarity) on the 64-dim state vectors.
   - L2-normalize all vectors before insertion: `faiss.normalize_L2(vectors)`
   - Index type: Flat (exact search) is fast enough for 5,000 vectors
3. **Exclusion zone**: When querying for analogues of day `t`, exclude the 252 most recent days (`t-252` to `t`) to prevent the LLM from seeing near-identical recent windows that might leak forward information or cause recency bias.
4. **Query protocol**: For day `t`, retrieve top-10 nearest neighbors by cosine similarity. Then cluster the 10 results into 3-5 distinct analogues using a simple rule: among the 10, greedily select the one with highest similarity, then skip any within 30 days of it, select the next highest, and repeat until 3-5 analogues are selected. This ensures temporal diversity in the analogues.

### Phase 3: Analogue Context Formatting (Week 2-3)

For each selected analogue window ending on day `t_a`, construct a structured context block:

```markdown
### Historical Analogue [N]: [Date Range]
**Similarity**: [cosine score, e.g., 0.92]
**State at the time**: VIX=[X], SPY trend=[Y], yield spread=[Z], breadth=[W]
**What the market looked like**: [2-3 sentence description derived from the state vector]

**What happened next (forward outcomes)**:
| Horizon | SPY | TLT | GLD | HYG | Best Sector | Worst Sector |
|---------|-----|-----|-----|-----|-------------|--------------|
| 5 days  | +X% | +Y% | +Z% | +W% | XLK +A%     | XLE -B%      |
| 10 days | ... | ... | ... | ... | ...         | ...          |
| 21 days | ... | ... | ... | ... | ...         | ...          |

**Regime ex-post**: [risk_on/risk_off/transition based on realized outcomes]
**Key lesson**: [1-sentence takeaway, e.g., "VIX at 22 with inverted curve led to 3 weeks of choppy sideways action; aggressive positioning was punished"]
```

The "key lesson" and "what the market looked like" descriptions are generated programmatically from the data, not by an LLM, to avoid compounding LLM hallucination risks.

### Phase 4: Prompt Integration (Week 3)

Modify the decision prompt template to include the analogues between the market data table and the instructions section:

```markdown
## Historical Analogues
The following periods had market conditions most similar to today.
Use these as empirical reference points, not as predictions.
Past performance does not guarantee future results.

[Analogue blocks inserted here]

**Analogue consensus**: [X of Y analogues were followed by positive SPY returns]
**Analogue dispersion**: [Range of outcomes: best case +A%, worst case -B% for SPY over 21 days]
```

### Phase 5: Evaluation (Week 3-4)

**A/B testing protocol** (simulated, not live):

Run the full pipeline (`build_context.py` -> LLM decision -> `execute_decision.py`) over the test period (2022-01-01 onward) under three conditions:

1. **Baseline**: Current system, no analogues. LLM sees only today's snapshot.
2. **RAG-3**: LLM sees today's snapshot plus 3 historical analogues (no similarity filter).
3. **RAG-3-filtered**: LLM sees today's snapshot plus 3 historical analogues, but only if the top analogue has cosine similarity >= 0.88. On days where no analogue passes the threshold, the LLM receives a note: "No sufficiently similar historical period found. Exercise additional caution."

**Metrics to compare across conditions**:

| Metric | Baseline | RAG-3 | RAG-3-filtered |
|--------|----------|-------|----------------|
| Annualized return | Measure | Measure | Measure |
| Sharpe ratio | Measure | > baseline | > baseline |
| Sortino ratio | Measure | > baseline | > baseline |
| Max drawdown | < 15% | < 15% | < 15% |

| Conviction calibration (Spearman) | ~0.05 | > 0.15 | > 0.20 |
| Expected Calibration Error (ECE) | Measure | < baseline | < baseline |

**Calibration details**: ECE bins convictions into 3 levels (high=0.9, medium=0.6, low=0.3) and compares stated confidence to actual profitability rate. Brier score = mean((confidence - outcome)^2). Conviction-Sharpe: Spearman correlation between conviction level and forward 21-day Sharpe per trade.

### Phase 6: Ablation Studies (Week 4)

1. **Number of analogues**: RAG-1, RAG-3, RAG-5, RAG-10 (information overload vs. insufficient context)
2. **Similarity threshold**: 0.80, 0.85, 0.88, 0.90, 0.95 (noisy analogues vs. too few triggered days)
3. **State vector dimensions**: 32-dim (no rolling avgs), 64-dim (full), 128-dim (per-sector breakdowns)
4. **Exclusion zone**: 63, 126, 252, 504 days (recency bias vs. analogue availability)
5. **Forward outcome horizon**: 5-day only, 21-day only, or both (current design)

## Literature Review

### Core Papers

1. **Ye & Durrett (2022)** -- "The Unreliability of Explanations in Few-Shot Prompting for Textual Reasoning." *NeurIPS 2022*.
   - **Extract**: Demonstrates that LLMs can produce correct answers from in-context examples even when the explanations in those examples are wrong. This has a critical implication for our design: the "key lesson" text in each analogue matters less than the raw data (forward returns table). Design the analogues to foreground data over narrative. Also: more examples is not always better -- there is an optimal count beyond which noise dominates.

2. **Johnson, Douze & Jegou (2017)** -- "Billion-Scale Similarity Search with GPUs." *arXiv:1702.08734*.
   - **Extract**: FAISS library design, index types, and performance characteristics. For our use case (5,000 vectors, 64 dimensions), `IndexFlatIP` with L2-normalized vectors provides exact cosine similarity search in <1ms. No need for approximate methods (IVF, HNSW) at this scale.

3. **Guo, Pleiss, Sun & Weinberger (2017)** -- "On Calibration of Modern Neural Networks." *ICML 2017*.
   - **Extract**: Formal definition of calibration, Expected Calibration Error (ECE) metric, reliability diagrams. Temperature scaling as a post-hoc calibration method. Apply their ECE computation to measure whether the LLM's conviction levels are well-calibrated to actual trade outcomes.

4. **Lo, Mamaysky & Wang (2000)** -- "Foundations of Technical Analysis: Computational Algorithms, Statistical Inference, and Empirical Implementation." *Journal of Finance*, 55(4), 1705-1765.
   - **Extract**: Formalizes the idea that recognizable patterns in financial data carry information. Their kernel regression approach to pattern matching provides theoretical grounding for our nearest-neighbor approach. Key finding: technical patterns have conditional return distributions that differ from unconditional -- this is precisely the information we want the analogues to convey to the LLM.

### Supporting Papers

5. **Lewis, Perez, Piktus et al. (2020)** -- "Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks." *NeurIPS 2020*.
   - **Extract**: The RAG framework (retrieve, concatenate, generate). Our "documents" are historical market windows, our "query" is today's state, and "generation" is the trade decision.

6. **Min, Lyu, Holtzman et al. (2022)** -- "Rethinking the Role of Demonstrations in In-Context Learning." *EMNLP 2022*.
   - **Extract**: The format and distribution of in-context examples matters more than their correctness. Structured historical context with accurate data activates reasoning capabilities regardless of whether analogues are truly "most similar."

7. **Brown, Mann, Ryder et al. (2020)** -- "Language Models are Few-Shot Learners." *NeurIPS 2020 (GPT-3)*.
   - **Extract**: Foundational evidence that LLMs perform tasks from few examples without gradient updates. Our approach is few-shot learning applied to trading: "here are similar past conditions and outcomes -- decide about today."

8. **Bailey & Lopez de Prado (2014)** -- "The Deflated Sharpe Ratio." *Journal of Portfolio Management*, 40(5).
   - **Extract**: Required for all strategy comparisons. Must account for ~15 total comparisons (3 conditions + ablation variants).

- **Primary**: RAG-3-filtered achieves Deflated Sharpe Ratio > 0.90 with Sharpe improvement >= 0.10 over baseline

- Analogue retrieval adds > 500ms latency per decision (should be <5ms for FAISS on 5k vectors, but prompt rendering and LLM processing of longer prompts could add up)

## Anti-Overfitting Warnings

### Critical Risks

1. **Zero trainable parameters does NOT mean zero overfitting risk**: The state vector design (64 features), similarity threshold (0.88), number of analogues (3), and exclusion zone (252 days) are all design choices that could be optimized against test-set performance. **Mitigation**: Fix all design parameters before seeing any test-set results. Document domain-knowledge rationale for each. Count each ablation variant in the Deflated Sharpe Ratio.

2. **Analogue cherry-picking by the LLM**: The LLM may selectively attend to whichever analogue confirms its pre-existing bias (anchoring). **Mitigation**: Include the "analogue consensus" summary (majority vote) as a pre-computed aggregate. Prompt the LLM to weigh consensus over individual analogues. Monitor whether regime assessments correlate with consensus or with cherry-picked individuals.

3. **Non-stationarity of analogue relevance**: A 2010s analogue may not apply in the 2020s due to structural changes (ZIRP, crypto, post-COVID). **Mitigation**: Track average analogue "freshness." Consider recency-weighted similarity: `adjusted_sim = cosine_sim * 0.95^(years_ago)`.

4. **Prompt length and attention dilution**: 3-5 analogue blocks add ~1,500-2,500 tokens. **Mitigation**: Keep blocks compact (table-heavy, narrative-light). Place analogues after current data. Verify individual-asset signal quality does not degrade with longer prompts.

5. **Lookback contamination**: State vector normalization uses expanding windows including recent data. **Mitigation**: Inherent to expanding-window design and acceptable. Document approach and verify state vector distribution stability across time.

6. **Analogue quality verification**: Cosine similarity in 64-dim space can be high even for superficially different markets. **Mitigation**: Qualitative audit of 20 random retrievals. If > 30% feel wrong, redesign the state vector.

7. **Forward outcome data leakage**: Ensure "regime_expost" labels are computed from 21-day forward returns using raw price data only, with no model dependency. The forward returns for a 2015 analogue end in 2015 -- no future data is used.

8. **LLM version sensitivity**: Results with one Claude version may not transfer to another. **Mitigation**: Record exact model version. Run a subset of tests with two versions if possible. This is an inherent limitation of LLM-in-the-loop systems.
