## System Context

This research targets **llm-quant**, a systematic macro paper-trading system managing a $100,000 portfolio across 39 assets spanning six asset classes:

- **US equity ETFs** (17): SPY, QQQ, IWM, DIA, VTI, IVV, XLK, XLF, XLE, XLV, XLI, XLY, XLP, XLU, XLRE, XLB, XLC
- **International equity** (3): VGK, EEM, EFA
- **Fixed income** (6): TLT, IEF, SHY, LQD, HYG, TIP
- **Commodities** (3): GLD, SLV, USO
- **Crypto** (5): BTC-USD, ETH-USD, SOL-USD, XRP-USD, ADA-USD
- **Forex** (5): EURUSD=X, GBPUSD=X, USDJPY=X, AUDUSD=X, USDCHF=X

**Technology stack**: Python 3.12, Polars DataFrames, DuckDB persistent storage, Pydantic configs, Jinja2 prompt templates. The system runs as a CLI (`pq`) with a Claude LLM integration for trade decisions.

**Current regime detection** (file: `src/llm_quant/brain/context.py`, functions `_get_vix()` and `_get_yield_spread()`):
- VIX level fetched from `^VIX` in DuckDB (`market_data_daily` table)
- Yield spread approximated as `(TLT/SHY - 1.0) * 100` -- a price-ratio proxy, not actual yield data
- SPY trend from SMA50 vs SMA200 in `_get_spy_trend()`
- Three regime labels only: `risk_on`, `risk_off`, `transition` (defined in `src/llm_quant/brain/models.py`, class `MarketRegime(StrEnum)`)
- No credit spread data, no dollar dynamics, no transition probabilities, no regime confidence calibration

**Current indicator pipeline** (file: `src/llm_quant/data/indicators.py`, function `compute_indicators()`):
- SMA(20, 50, 200), Bollinger Bands(20, 2), RSI(14), MACD(12, 26, 9), ATR(14)
- All computed per-symbol via Polars `over("symbol")` partitioning
- No macro-level indicators (credit spreads, dollar index, FRED data)

**Pipeline flow**: `scripts/build_context.py` fetches data if stale, computes indicators, builds `MarketContext` dataclass, renders Jinja2 prompt, outputs JSON to stdout. Claude LLM reads the JSON and outputs `TradingDecision` with signals. `scripts/execute_decision.py` feeds signals through `RiskManager.filter_signals()` (8 pre-trade checks in `src/llm_quant/risk/manager.py`) and executes approved trades.

**Risk limits** (file: `src/llm_quant/risk/limits.py`): max 2% NAV per trade, 10% per position (5% crypto, 8% forex), gross exposure < 200%, net < 100%, sector concentration < 30%, cash reserve >= 5%, stop-loss required, max 15% drawdown.

**Performance targets**: Sharpe > 0.8, max drawdown < 15%, Sortino > 1.0, Calmar > 0.5, annualized return 8-15%.

## Research Question

Does adding investment-grade credit spreads (IG OAS) and the US dollar index (DXY) as co-equal regime factors alongside VIX and the yield curve produce a Hidden Markov Model that delivers higher out-of-sample Sharpe ratio and lower maximum drawdown compared to the current two-factor heuristic regime classifier?

Specifically: can a 4-state Gaussian HMM trained on weekly observations of {VIX, 10Y-2Y slope, IG OAS, DXY 3-month momentum} correctly identify credit stress episodes (Q4 2018, early 2020, early 2022) 2-4 weeks before they register in equity volatility alone?

---

## Hypothesis

**H1 (primary)**: The current VIX + TLT/SHY proxy misclassifies approximately 20-30% of trading weeks because it misses two critical macro channels: credit availability (spread widening often leads equity vol by 2-4 weeks) and dollar strength (a strong dollar compresses EM/commodity returns and signals risk aversion in global capital flows). A 4-factor HMM with states {`risk_on`, `credit_stress`, `dollar_squeeze`, `risk_off`} will:

- Reduce false `risk_on` labels during credit deterioration by >= 50% (measured against realized forward 1-month equity drawdown > 5%)
- Improve out-of-sample Sharpe ratio by >= 0.15 (from roughly 0.6 to 0.75+) on a simple regime-following allocation strategy
- Reduce maximum drawdown by >= 3 percentage points on the test set

**H2 (secondary)**: The 4-state model produces economically interpretable regimes -- each state maps to a distinct allocation template, and state-conditional return distributions are statistically separable (multivariate Hotelling T-squared test, p < 0.01).

**Null hypothesis**: The additional factors add noise without improving classification accuracy; the 2-factor heuristic is sufficient for a 39-asset paper portfolio with daily rebalancing frequency.

**Expected magnitude**: The largest gains will appear during credit stress episodes (e.g., Feb 2018 VIX spike, Q4 2018 Fed tightening, March 2020 COVID, Q1-Q2 2022 rate shock), where the current model is slowest to react. Expect 15-25% improvement in regime accuracy during these periods specifically.

## Methodology

### Phase 1: Data Acquisition and Preprocessing (Week 1)

1. **Build a FRED data fetcher** (`src/llm_quant/data/macro_fetcher.py`) using the `fredapi` Python library (free API key from https://fred.stlouisfed.org/docs/api/api_key.html). Fetch daily series for:
   - `BAMLC0A0CM` -- ICE BofA US Corporate Index Option-Adjusted Spread (IG OAS)
   - `BAMLH0A0HYM2` -- ICE BofA US High Yield Index Option-Adjusted Spread (HY OAS, for validation)
   - `DTWEXBGS` -- Nominal Broad US Dollar Index (trade-weighted)
   - `T10Y2Y` -- 10-Year minus 2-Year Treasury Constant Maturity spread
   - `T10Y3M` -- 10-Year minus 3-Month Treasury spread (cross-validation)
   - `VIXCLS` -- CBOE VIX close (daily, consistent source vs. Yahoo's ^VIX)
   - `DCOILWTICO` -- WTI crude oil spot price (supplementary)

2. **Resample to weekly frequency** (Friday close) to reduce noise and match the decision horizon (the portfolio rebalances at most daily, but regime changes are macro phenomena operating on weekly-to-monthly timescales).

3. **Construct 4 feature columns**:
   - `vix_level`: VIXCLS weekly close (raw level, not return)
   - `yield_slope`: T10Y2Y weekly close (basis points)
   - `ig_oas`: BAMLC0A0CM weekly close (basis points)
   - `dxy_mom_13w`: 13-week percentage change in DTWEXBGS (momentum, not level, to capture dollar trend direction)

4. **Winsorize** at 1st/99th percentile to limit outlier influence. **Standardize** each feature to zero mean, unit variance using the training set's statistics only.

5. **Split**: Training set 2005-01-01 to 2018-12-31 (14 years, ~728 weekly observations). Test set 2019-01-01 to 2025-12-31 (7 years, ~364 observations). The split date is chosen because it precedes both the 2019 rate-cut pivot and COVID, providing genuine out-of-sample stress events.

### Phase 2: HMM Estimation (Week 2)

1. **Install `hmmlearn`** (BSD-licensed, scikit-learn compatible). Use `GaussianHMM` with:
   - `n_components=4` (matching the hypothesized 4 regimes)
   - `covariance_type="full"` (allows factor correlations to differ across states)
   - `n_iter=200`, `tol=1e-4`
   - 2 lags of each feature appended as additional columns (so 12 features total: 4 current + 4 lag-1 + 4 lag-2), capturing momentum in regime transitions

2. **Multiple restarts**: Run 50 random initializations, select the model with highest log-likelihood. This mitigates the EM algorithm's sensitivity to starting conditions.

3. **Model selection**: Also train 3-state and 5-state models. Compare using BIC (Bayesian Information Criterion) to determine optimal state count without overfitting.

4. **Label regimes**: After training, inspect per-state emission means to assign economic labels:
   - State with low VIX + steep yield curve + tight IG OAS + neutral DXY = `risk_on`
   - State with moderate VIX + widening IG OAS + any slope = `credit_stress`
   - State with rising DXY momentum + tightening financial conditions = `dollar_squeeze`
   - State with high VIX + flat/inverted curve + wide IG OAS = `risk_off`

5. **Transition matrix analysis**: Extract and interpret the 4x4 transition probability matrix. Compute expected regime durations (1 / (1 - p_ii)). Validate that `risk_on` is the most persistent state and that `credit_stress` often precedes `risk_off`.

### Phase 3: Baseline Comparison (Week 3)

1. **Replicate the current 2-factor heuristic**: Implement the existing VIX + TLT/SHY logic in a standalone function with the same weekly frequency. Generate regime labels for the full sample.

2. **Build a regime-following allocation strategy** for both models:
   - `risk_on`: 60% equities (SPY), 20% commodities (GLD), 10% EM (EEM), 10% cash
   - `credit_stress`: 20% equities, 30% TLT, 30% cash, 10% GLD, 10% SHY
   - `dollar_squeeze`: 10% equities, 20% GLD, 20% TLT, 20% TIP, 30% cash
   - `risk_off`: 0% equities, 40% TLT, 10% GLD, 50% cash

3. **Backtest both models** on the test set (2019-2025) with:
   - Weekly rebalancing (match regime frequency)
   - 10 bps transaction costs per leg
   - 1-week implementation lag (trade on Monday close after Friday regime signal)

4. **Compute metrics**: Sharpe ratio, Sortino ratio, Calmar ratio, max drawdown, annualized return, annualized volatility, regime accuracy vs. realized forward returns, confusion matrix of regime labels vs. ex-post optimal labels.

### Phase 4: Integration Design (Week 4)

1. **Design the context enrichment**: Add `credit_stress_score` (0.0-1.0 continuous), `dollar_regime` (string label), and `regime_confidence` (HMM posterior probability of most likely state) to the JSON output of `build_context.py`.

2. **Write configuration**: `config/regime_rules.toml` mapping regime states to allocation tilts and risk parameter adjustments.

3. **Document the upgrade path**: Specify exactly which functions in `context.py` change and what new fields appear in `MarketContext`.

## Literature Review

### Primary References

1. **Estrella & Hardouvelis (1991)**, "The Term Structure as a Predictor of Real Economic Activity," *Journal of Finance*, 46(2), 555-576.
   - **Extract**: The statistical relationship between yield curve slope and recession probability. Use their probit specification as the economic foundation for including T10Y2Y as a regime factor. Their finding that an inverted curve predicts recession 4-6 quarters ahead justifies the yield curve's role as a leading indicator in the HMM observation vector.

2. **Ang & Bekaert (2002)**, "Regime Switches in Interest Rates," *Journal of Business & Economic Statistics*, 20(2), 163-182.
   - **Extract**: Methodology for fitting regime-switching models to yield curve data. Their two-state model (high/low volatility) for short rates provides the template for extending to a 4-state model across multiple macro factors. Key insight: transition probabilities vary over time -- consider robustness of a time-homogeneous HMM.

3. **Ferson & Harvey (1991)**, "The Variation of Economic Risk Premiums," *Journal of Political Economy*, 99(2), 385-415.
   - **Extract**: Evidence that credit spreads and term spreads predict time-varying expected returns across asset classes. This paper provides the theoretical justification for including IG OAS alongside the yield curve: both carry independent information about the macroeconomic state.

4. **Lopez de Prado (2018)**, *Advances in Financial Machine Learning*, Wiley, Chapter 17: Bet Sizing.
   - **Extract**: The meta-labeling framework for calibrating position sizes to model confidence. Apply the regime posterior probability from the HMM as an input to bet sizing -- higher confidence in `risk_on` permits larger equity allocations, while low-confidence mixed states warrant reduced exposure. Also extract Chapter 7's cross-validation methodology for time-series data (purged k-fold).

5. **Mueller, Tahbaz-Salehi & Vedolin (2017)**, "Exchange Rates and Monetary Policy Uncertainty," *Journal of Finance*, 72(3), 1213-1252.
   - **Extract**: The mechanism by which dollar strength transmits monetary policy shocks across borders. Their finding that DXY momentum reflects global risk appetite justifies including it as a regime factor independent of VIX.

### Supplementary References

6. **Hamilton (1989)**, "A New Approach to the Economic Analysis of Nonstationary Time Series and the Business Cycle," *Econometrica*, 57(2), 357-384.
   - **Extract**: The foundational paper on Markov-switching models in macroeconomics. Use as methodological reference for the HMM specification and EM estimation procedure.

7. **Guidolin & Timmermann (2007)**, "Asset Allocation under Multivariate Regime Switching," *Journal of Economic Dynamics and Control*, 31(11), 3503-3544.
   - **Extract**: Extension of regime-switching models to multi-asset allocation. Their finding that regime-aware portfolios significantly outperform static allocations provides the economic motivation for this research.

8. **Kritzman, Page & Turkington (2012)**, "Regime Shifts: Implications for Dynamic Strategies," *Financial Analysts Journal*, 68(3), 22-39.
   - **Extract**: Practical implementation guidance for regime-based allocation in institutional portfolios. Their Mahalanobis distance approach to regime identification serves as an alternative/validation method against the HMM.

## Anti-Overfitting Warnings

### Specific Pitfalls for This Research

1. **In-sample regime labeling bias**: When inspecting HMM states to assign economic labels, there is a temptation to cherry-pick the label assignment that maximizes in-sample performance. **Mitigation**: Label states using training-set emission parameters only. Verify that the same labeling holds on the test set without modification.

2. **Data snooping on the train/test split**: The 2018/2019 split was chosen because it creates a clean pre/post boundary. But this choice itself embeds knowledge of when major regime changes occurred. **Mitigation**: Run robustness checks with alternative splits (2016/2017, 2017/2018, 2019/2020). Report sensitivity of results to the split date.

3. **HMM state count selection**: Testing 3, 4, and 5 states and picking the best performer on the test set is a form of overfitting to the test set. **Mitigation**: Select state count using BIC on the training set only. Report test-set results for all three specifications, not just the BIC-optimal one.

4. **Look-ahead bias in weekly aggregation**: Using Friday close for features but allowing trades on Friday close creates a simultaneity problem. **Mitigation**: All trades use the following Monday close. Feature construction uses data available by Thursday's close at the latest.

5. **Survivorship bias in FRED series**: The IG OAS series (BAMLC0A0CM) was revised in 2017 when ICE acquired the BofA indices. Historical data before the revision may have different construction methodology. **Mitigation**: Document any known methodology changes. Test whether model performance differs pre/post 2017 revision.

6. **Overfitting the allocation templates**: Optimizing the regime-to-allocation mapping on the test set defeats the purpose. **Mitigation**: Fix the allocation templates before running the test-set backtest. Use simple, economically motivated weights (not optimized). If optimization is desired, use only the training set with walk-forward validation.

7. **Regime persistence assumption**: The HMM assumes a time-homogeneous transition matrix (transition probabilities don't change over time). If the macro environment fundamentally shifted (e.g., post-COVID monetary policy), the training-set transition matrix may not apply. **Mitigation**: Compare transition matrices estimated on different sub-periods. Report whether a time-varying transition probability model (using the `statsmodels` Markov switching regression) materially changes results.

8. **Small test set**: 7 years of weekly data is only ~364 observations, containing perhaps 3-4 genuine regime transitions. Bootstrap confidence intervals are essential -- do not rely on point estimates alone. **Mitigation**: Use block bootstrap (block size = 13 weeks) to preserve autocorrelation when constructing confidence intervals for Sharpe and drawdown.
