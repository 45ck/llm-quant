# 4.2 Sparse Autoencoder Factors

> **Domain**: Machine Learning & Deep Learning | **Priority**: P3 | **Status**: Pending

## System Context

This research targets **llm-quant**, a systematic macro trading system managing a $100k paper portfolio across 39 tradeable assets spanning six asset classes:

- **US Equity ETFs (17)**: SPY, QQQ, IWM, DIA, VTI, IVV, XLK, XLF, XLE, XLV, XLI, XLY, XLP, XLU, XLRE, XLB, XLC
- **International Equity (3)**: VGK, EEM, EFA
- **Fixed Income (6)**: TLT, IEF, SHY, LQD, HYG, TIP
- **Commodities (3)**: GLD, SLV, USO
- **Crypto (5)**: BTC-USD, ETH-USD, SOL-USD, XRP-USD, ADA-USD
- **Forex (5)**: EURUSD=X, GBPUSD=X, USDJPY=X, AUDUSD=X, USDCHF=X

**Stack**: Python 3.12, Polars DataFrames, DuckDB persistent storage, Jinja2 prompt templates.

**Current indicator pipeline**: The function `compute_indicators()` in `src/llm_quant/data/indicators.py` computes per-asset technical indicators: SMA(20), SMA(50), SMA(200), Bollinger Bands (20,2), RSI(14), MACD(12,26,9), and ATR(14). Each indicator is computed independently per symbol via Polars `over("symbol")` partitioning. There is no cross-asset indicator -- every signal treats each asset in isolation.

**Current LLM context**: The Jinja2 template `config/prompts/trader_decision.md` renders a table of all 39 assets with columns `[Symbol, Close, Chg%, SMA20, SMA50, RSI14, MACD, ATR14, Vol]`. The LLM must mentally compute cross-asset patterns (e.g., "XLK RSI is high while XLE RSI is low, suggesting rotation from tech to energy"). This is implicit and unreliable -- the LLM has no structured cross-sectional signal.

**Feature count**: Each asset has 15 features available (OHLCV + the 11 indicators). For 39 assets, the cross-sectional feature vector on any given day is 39 x 15 = 585 dimensions. With ~5,000 trading days, the cross-sectional sample is severely limited relative to feature dimensionality, making regularization essential.

**Decision pipeline**: `scripts/build_context.py` assembles the context including portfolio state from `src/llm_quant/trading/portfolio.py`, indicator data via `src/llm_quant/brain/context.py::build_market_context()`, and macro indicators (VIX, yield spread, SPY trend). The LLM's JSON response is parsed by `src/llm_quant/brain/parser.py::parse_trading_decision()` into a `TradingDecision` containing `TradeSignal` objects, which pass through 8 risk checks in `src/llm_quant/risk/manager.py::RiskManager.filter_signals()` before execution.

**Targets**: Sharpe > 0.8, max drawdown < 15%, Sortino > 1.0, Calmar > 0.5.

## Research Question

Does a sparse autoencoder (SAE) trained on the 39-asset daily feature matrix discover latent cross-asset factors with return-predictive information beyond what individual-asset SMA/RSI/MACD indicators provide? Specifically:

1. Do the learned sparse factors correspond to interpretable economic structures (sector rotation, flight-to-quality, dollar strength, liquidity stress)?
2. Do the top-K most active factor features on a given day predict the cross-section of next-day or next-week asset returns?
3. Does augmenting the LLM's context with sparse factor activations improve trade decision quality compared to raw indicator tables alone?

## Hypothesis

Individual per-asset indicators ignore the cross-sectional structure of the 39-asset universe. A sparse autoencoder with an overcomplete dictionary (input 585, hidden 1024) and L1 penalty will learn nonlinear latent factors that capture:

- **Sector rotation momentum**: When tech-sector indicators collectively strengthen while energy weakens, a single factor activation should capture this pattern more cleanly than 17 separate RSI values.
- **Flight-to-quality dynamics**: Correlated movements in TLT/IEF/GLD up + SPY/QQQ/HYG down should activate a "risk-off" factor.
- **Currency/commodity linkages**: Dollar strength (USDJPY/USDCHF up) coinciding with commodity weakness (GLD/SLV/USO down) should map to a "strong dollar" factor.
- **Liquidity stress**: HYG-LQD spread widening + VIX spike + declining volumes across equities should activate a "stress" factor.

**Prediction**: The top-10 most active SAE factors on day `t` will have Spearman rank correlation > 0.15 with the cross-sectional ranking of asset returns on day `t+1` to `t+5`, compared to < 0.05 for the best individual indicator (RSI or MACD) alone.

**Null hypothesis**: SAE factor activations have no statistically significant predictive power for forward cross-sectional returns beyond individual indicators (Deflated Sharpe Ratio < 0.5 for any factor-based signal).

## Methodology

### Phase 1: Feature Matrix Construction (Week 1)

1. **Extract daily feature vectors** from DuckDB. For each trading day `t`, construct a 585-dimensional vector by concatenating 15 features for each of 39 assets (in fixed symbol order):
   - Features per asset: `[close_norm, log_return_1d, log_return_5d, sma20_ratio, sma50_ratio, sma200_ratio, bb_pct_b, rsi_14_norm, macd_norm, macd_hist_norm, atr_14_norm, volume_zscore, high_low_range, close_to_high, close_to_low]`
   - Normalization: All features standardized cross-sectionally (z-score across assets) per day, then clipped to [-3, 3] to limit outlier influence.
2. **Handle missing data**: Assets with missing data on a given day (crypto before 2017, some forex) are zero-filled after z-scoring (i.e., treated as "average"). Track a binary mask of which assets are present per day.
3. **Final matrix**: Shape `(num_days, 585)` as a Polars DataFrame saved to DuckDB and exported as NumPy.

### Phase 2: Sparse Autoencoder Training (Week 2)

**Architecture**:
```
Encoder:
  Input: x in R^585
  -> Linear(585, 1024) + ReLU  # overcomplete expansion
  -> L1 penalty on activations (lambda * ||h||_1)

Decoder:
  -> Linear(1024, 585)  # reconstruction

Loss = MSE(x, x_hat) + lambda * ||h||_1
```

The overcomplete dictionary (585 -> 1024) is intentional, following Bricken et al. (2023). With strong L1 regularization, only a small subset of the 1024 hidden units will be active for any given input, producing a sparse code that decomposes the input into interpretable "atomic" factors.

**Training protocol**:
- **Train/test split**: Train on data before 2020-01-01. Validate on 2020-01-01 to 2021-12-31. Test on 2022-01-01 onward.
- **L1 coefficient search**: lambda in {0.001, 0.003, 0.01, 0.03, 0.1}. Select lambda that yields average sparsity of 5-15% (50-150 active features per input) on the validation set. Fix this value before computing any test-set metrics.
- **Optimizer**: Adam, lr=1e-3, batch_size=64, epochs=500, early stopping on validation reconstruction loss (patience=30).
- **Dictionary normalization**: After each gradient step, normalize decoder columns to unit L2 norm (prevents the encoder from shrinking activations while the decoder compensates with large weights).

### Phase 3: Factor Interpretation (Week 3)

1. **Activation analysis**: For each of the 1024 hidden units, identify which input features most strongly activate it (top-5 by mean activation on the training set). Group factors into interpretable themes:
   - Which assets load most heavily on each factor?
   - Do factors align with known economic structures (sector, asset class, geography)?
2. **Dead feature detection**: Hidden units active on < 1% of training samples are "dead" and should be excluded from analysis.
3. **Factor correlation with known signals**:
   - Compute Spearman correlation between each factor activation and: VIX level, 10Y-2Y spread, HYG-LQD spread, DXY proxy (average of USDJPY/USDCHF), SPY momentum
   - Factors highly correlated with known signals are less valuable (they rediscover what we already have)
4. **Novel factor identification**: The most valuable factors are those with low correlation to all existing indicators but high reconstruction importance (high decoder weight norm). These capture cross-sectional structure invisible to per-asset indicators.

### Phase 4: Predictive Evaluation (Week 4)

**Cross-sectional return prediction**:
1. For each test day `t`, extract the SAE activation vector `h_t` (1024-dim, sparse).
2. Use the top-K (K=10, 20, 50) most active features as predictors for next-day and next-week cross-sectional returns.
3. **Model**: Ridge regression (alpha determined by inner CV on training data) predicting `r_{t+1}` from `h_t[top_K]`. No nonlinear models to avoid overfitting on small sample.
4. **Metric**: Rank IC (Spearman correlation between predicted and realized cross-sectional return ranks) averaged over test days.

**Two integration paths (evaluate both)**:

**Path A -- LLM Context Augmentation**:
- Add a "Cross-Asset Factors" section to `config/prompts/trader_decision.md` showing the top-5 most active factors today with their interpretable labels and activation strengths
- The LLM uses these as additional signal for its decision
- Evaluate by comparing trade decision quality (Sharpe, Sortino) with and without factor context

**Path B -- Ridge Regression Ensemble Overlay**:
- Train a separate Ridge regression model: `position_size_adjustment = Ridge(h_t[top_K])` predicting next-5d asset returns
- If Ridge prediction agrees with LLM signal direction: multiply position size by 1.25 (+25%)
- If Ridge prediction disagrees with LLM signal direction: multiply position size by 0.75 (-25%)
- This creates a simple ensemble where the quantitative model provides a confidence adjustment on the LLM's qualitative judgment
- Hard constraint: adjusted position size still must pass all 8 risk checks in `RiskManager`

## Literature Review

### Core Papers

1. **Bricken, Templeton, Batson et al. (2023)** -- "Towards Monosemanticity: Decomposing Language Models With Dictionary Learning." *Anthropic Technical Report*.
   - **Extract**: Overcomplete sparse autoencoders decompose polysemantic neural network activations into monosemantic (single-concept) features. Their L1 penalty calibration methodology (targeting specific sparsity levels), decoder weight normalization trick, and dead feature resampling technique. While their application is to language models, the mathematical framework (overcomplete dictionaries with L1 regularization) applies identically to financial feature decomposition. Their finding that interpretability improves with overcompleteness motivates our 585->1024 expansion.

2. **Gu, Kelly & Xiu (2020)** -- "Empirical Asset Pricing via Machine Learning." *Review of Financial Studies*, 33(5), 2223-2273.
   - **Extract**: Comprehensive comparison of ML methods for cross-sectional return prediction. Their finding that the most important features are interaction terms (not individual characteristics) directly supports our SAE approach. Key benchmark: their out-of-sample R-squared values for various models on US equity returns. Their variable importance analysis methodology for interpreting learned factors.

3. **Rasekhschaffe & Jones (2019)** -- "Machine Learning for Stock Selection." *Financial Analysts Journal*, 75(3), 70-88.
   - **Extract**: Practical guidance on avoiding overfitting in ML-based stock selection. Their recommended pipeline of feature engineering -> dimensionality reduction -> penalized regression. Warning about the "100 features, 1000 stocks, 20 years" sample size trap -- directly relevant to our 585 features, 39 assets, 20 years scenario.

4. **Feng, Polson & Xu (2018)** -- "Deep Learning in Characteristics-Based Asset Pricing." *SSRN Working Paper*.
   - **Extract**: Architecture for deep factor models in asset pricing. Their conditional autoencoder approach where factor loadings are conditioned on asset characteristics. Comparison of linear PCA factors vs. nonlinear autoencoder factors. Evidence that nonlinear factors capture momentum-reversal interaction effects invisible to linear methods.

5. **Freyberger, Neuhierl & Weber (2020)** -- "Dissecting Characteristics Nonparametrically." *Review of Financial Studies*, 33(5), 2326-2377.
   - **Extract**: Nonparametric approach to understanding which characteristics matter for asset returns and how they interact. Their adaptive group LASSO technique for feature selection with theoretical guarantees. Evidence that many "significant" characteristics are redundant once nonlinear interactions are allowed -- motivating sparse (not dense) factor models.

### Supporting Papers

6. **Bailey & Lopez de Prado (2014)** -- "The Deflated Sharpe Ratio." *Journal of Portfolio Management*, 40(5), 94-107.
   - **Extract**: Deflated Sharpe Ratio formula for all strategy evaluations.

7. **Olshausen & Field (1996)** -- "Emergence of Simple-Cell Receptive Field Properties by Learning a Sparse Code for Natural Images." *Nature*, 381, 607-609.
   - **Extract**: The foundational work showing that sparse overcomplete codes learn interpretable features from data. Theoretical basis for why sparsity + overcompleteness yields monosemantic (single-concept) features.

8. **Kelly, Pruitt & Su (2019)** -- "Characteristics Are Covariances: A Unified Model of Risk and Return." *Journal of Financial Economics*, 134(3), 501-524.
   - **Extract**: Instrumental PCA (IPCA) as a linear baseline for latent factor extraction from asset characteristics. Their framework for evaluating whether learned factors explain cross-sectional return variation. Use as a comparison benchmark against the SAE.

## Evaluation Criteria

### Success Thresholds
- **Primary**: Mean rank IC (5-day horizon) on the test set > 0.10 (significant at 5% level via Newey-West t-statistic with lag=5)
- **Primary**: Deflated Sharpe Ratio for the best integration path (A or B) exceeds 0.95
- **Secondary**: At least 5 factors have interpretable labels matching known economic structures and are not redundant with existing indicators (correlation < 0.5 with VIX, yield spread, or SPY momentum)
- **Secondary**: Portfolio Sharpe improvement of >= 0.05 over indicator-only baseline

### Failure Conditions
- SAE reconstruction MSE on the test set exceeds 2x training MSE (model does not generalize)
- Fewer than 20 non-dead features (extreme sparsity, underfitting) or more than 500 active features per sample (insufficient sparsity)
- All top-10 predictive factors have > 0.7 correlation with VIX or SPY returns (SAE just rediscovered known signals)
- Rank IC on the test set is not statistically different from zero (Newey-West p > 0.10)
- Deflated Sharpe Ratio < 0.5 for all variants

### Comparison Benchmarks
1. **Per-asset indicators only**: Current system with no cross-sectional signal
2. **PCA factors**: Linear PCA on the same 585-dim feature matrix, top-10 components
3. **IPCA factors** (Kelly et al. 2019): Instrumented PCA as the state-of-the-art linear benchmark
4. **Sector-average indicators**: Simple cross-sectional signal (average RSI per sector, average momentum per sector)

## Anti-Overfitting Warnings

### Critical Risks

1. **Extreme feature-to-sample ratio**: 585 input features with ~3,700 training samples gives a ratio of 6.3:1. Even with L1 regularization reducing effective dimensionality, the autoencoder's 585x1024 + 1024x585 = ~1.2M parameters vastly exceed the sample count. **Mitigation**: (a) The L1 penalty must be strong enough to yield sparsity < 15%. (b) Monitor reconstruction MSE gap between train and validation closely. (c) Report the effective number of parameters (non-zero decoder weights) alongside all results. (d) Consider reducing hidden dim to 512 if validation loss diverges.

2. **Walk-forward requirement**: A single train/test split can be misleading if the test period is structurally unusual (e.g., 2022 was a rare simultaneous equity+bond selloff). **Mitigation**: Report results on the held-out test set (2022+) as the primary metric, but also conduct walk-forward cross-validation with 252-day train / 63-day test rolling windows across the full sample. Results must be consistent across both evaluation approaches.

3. **Ridge regression degrees of freedom**: Using top-50 SAE features as Ridge inputs with 63-day test windows gives only 63/50 = 1.26 observations per parameter. **Mitigation**: Cap top-K at 10 for Ridge regression (10 free parameters per signal, matching the domain expert constraint). Use Ridge alpha from inner CV, not grid search on the test set.

4. **Normalization look-ahead bias**: Cross-sectional z-scoring requires the mean and standard deviation of all assets on day `t`. If computed using future data, this leaks information. **Mitigation**: Compute z-score statistics using only data available up to day `t`. In practice, use rolling 252-day expanding window statistics, not full-sample statistics.

5. **Factor interpretation p-hacking**: With 1024 factors, some will appear predictive by chance. If the top-10 by rank IC are selected on the test set, this is data snooping. **Mitigation**: Select the top-K factors by training-set rank IC. Evaluate the same fixed set on the test set. Report how many of the top-10 training-set factors remain in the top-20 test-set factors (stability metric). Apply Bonferroni correction when reporting individual factor significance.

6. **L1 coefficient selection bias**: Testing 5 values of lambda and selecting the best constitutes a minor multiple comparison. **Mitigation**: Use the Deflated Sharpe Ratio with `num_trials=5` (accounting for the 5 lambda values tested). Select lambda based on sparsity level (5-15%) on the validation set, not on downstream portfolio performance.

7. **SAE vs. simpler alternatives**: If PCA with 10 components achieves similar rank IC, the SAE adds complexity without benefit. **Mitigation**: PCA and IPCA are mandatory comparison benchmarks. The SAE must outperform both to justify its added complexity and training cost. If it does not, report the negative result and recommend the simpler approach.

8. **Path B gaming risk**: The Ridge overlay adjusts position sizes by +/-25%. If the Ridge model has poor calibration, this could systematically increase exposure in the wrong direction. **Mitigation**: Impose a minimum agreement threshold -- the overlay only fires when Ridge prediction confidence (absolute predicted return rank) exceeds the 60th percentile. Below that, no adjustment. Also verify that the overlay does not cause more risk check failures (i.e., the 25% increase does not push positions above the 10% weight limit in `check_position_weight()`).
